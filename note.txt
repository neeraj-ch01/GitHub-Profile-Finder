# Word Cloud Optimization and Stability Fixes

## Dependency Updates
- Replaced the legacy `react-wordcloud` package with `react-d3-cloud` (`^1.0.6`) in `package.json` and `package-lock.json`. This resolves fatal React 18 StrictMode crashes ("Cannot read properties of undefined (reading '0')") that were breaking the application when rendering the word cloud.

## Word Cloud Layout & Container Refactoring
- **Container Sizing**: Modified the wrapper element for the Word Cloud in `AnalyticsDashboard.js` from a dynamic `flex` container (which collapsed to 0x0 before measurement, rendering invisibly) to an explicit block display with `h-[400px] max-w-[400px] w-full mx-auto`. This establishes a firm, compact square boundary.
- **Canvas Scaling**: Set the internal `WordCloud` canvas width and height to explicitly match the container `400x400` aspect ratio.
- **Spiral Algorithm**: Transitioned the layout algorithm from the default archimedean spiral to `"rectangular"` to achieve optimal density and tight word packing within the square bounding box.

## Custom Rotation Rules Implemented
- The goal of these rules is to maximize layout density while ensuring multi-word topics remain readable.
- **Rule 1 (Long Words)**: Topics with a length greater than 6 characters or those containing a hyphen (`-`) are strictly enforced to render horizontally (`0°`).
- **Rule 2 (Small/Single Words)**: To prevent a sparse, cross-shaped word cloud caused by pushing all small words vertically, small words are now rotated using a deterministic string hashing function based on `charCodeAt`.
  - The algorithm calculates a hash sum of the characters.
  - Approximately **70%** of small words are rendered horizontally (`0°`).
  - The remaining **30%** are rendered vertically (`90°`).
  - This allows the `react-d3-cloud` layout engine to slot words together closely without long vertical words forcing wide padding gaps.
