fix: Broken Avatars

RCA (Root Cause Analysis)
Found 0 results: The GitHub API returns the total number of search results in a field called total_count. The backend serializes this exactly as total_count due to the @JsonProperty("total_count") annotation. However, SearchProfiles.js was expecting response-> totalCount. Since this was undefined, the frontend defaulted to 0 (using the fallback logic response.totalCount || 0). The 4 results were successfully mapped into the results array, but the counter always showed 0.
Broken Avatars: Similarly, the avatar link is provided by the backend as avatar_url. In SearchProfiles.js, the image src was bound to user.avatarUrl. Since user.avatarUrl was undefined, the browser couldn't fetch an image and instead displayed the alt text (the user's login handle) alongside a broken image icon.

Resolution
Updated the frontend components to correctly reference the snake_case properties expected from the backend payload.

Changes made:

In src/components/SearchProfiles.js:
Replaced response.totalCount with response.total_count.
Replaced user.avatarUrl with user.avatar_url.

In src/components/RepoDetail.js (preemptive fixes):
Replaced commitItem.author.avatarUrl with commitItem.author.avatar_url.
Replaced commitItem.htmlUrl and issue.htmlUrl with html_url.
