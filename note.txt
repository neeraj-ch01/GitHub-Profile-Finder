Enhance Markdown rendering, centralize API client, and fix UI data binding

This commit introduces several frontend improvements ranging from better 
Markdown display in repository details to improved API request logging.

- Added `apiClient.js` to provide a centralized Axios instance with 
  request and response interceptors for consistent frontend logging.
- Refactored `authenticationService`, `githubService`, and 
  `registrationService` to use the new `apiClient`.
- Implemented Mermaid.js integration to support flowchart and diagram 
  rendering within GitHub README files. Added responsive CSS rules 
  to prevent diagrams from distorting layout.
- Enhanced the Markdown renderer to resolve relative image URLs by 
  injecting the GitHub `download_url` for raw assets, fixing broken 
  images in repository READMEs.
- Fixed a data binding issue in `UserProfile.js` where `public_repos` 
  was incorrectly referenced as `publicRepos`.
- Added `mermaid` dependency to `package.json`.
