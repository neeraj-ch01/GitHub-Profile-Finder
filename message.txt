fix: add User-Agent header for GitHub API calls

1. The GithubRepoDTO was missing the owner field, and Jackson (the JSON parser) was throwing a serialization error because it was encountering properties from GitHub's API that it didn't know how to map.
2. The Spring WebClient was missing a User-Agent header, which the GitHub API strictly requires.

Changed done: 
- Added the missing owner field to GithubRepoDTO.java.
- Added the @JsonIgnoreProperties(ignoreUnknown = true) annotation to GithubRepoDTO, GithubSearchResponseDTO, and GithubUserProfileDTO so the backend won't fail if GitHub sends extra fields.
- Added a .defaultHeader("User-Agent", "GithubProfileFinderApp") to the WebClient in GithubSearchService.java to comply with GitHub's API requirements.
